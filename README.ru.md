## Обзор

Fetcho — лёгкое расширение для браузера Chromium (Manifest V3). Расширение работает как интеллектуальный посредник: вы вставляете ссылку, Fetcho открывает нужный сайт-загрузчик с уже вставленной ссылкой, а контент-скрипт нажимает кнопку загрузки. Никаких ручных переключений между вкладками и копирований между сайтами.

Из коробки поддерживаются шесть платформ.

| Платформа | Шаблоны ссылок | Перенаправляется на |
|---|---|---|
| Instagram | `instagram.com`, `instagr.am` | `igram.world` |
| Twitter / X | `twitter.com`, `x.com` | `ssstwitter.com` |
| YouTube Shorts | `youtube.com/shorts`, `youtu.be` | `ytshortsdl.io` |
| TikTok | `tiktok.com` | `ssstik.io` |
| Reddit | `reddit.com` | `rapidsave.com` |
| Facebook | `facebook.com`, `fb.com`, `fb.watch` | `fdown.net` |

> [!NOTE]
> Fetcho не хранит, не обрабатывает, не скачивает и не размещает видео-контент. Расширение лишь перенаправляет ссылки через общедоступные сторонние сервисы загрузки.

## Возможности

- **Автоматизация контент-скриптов** -- сайты-загрузчики заполняются и отправляются без участия пользователя
- **Автоопределение платформы** -- вставьте ссылку, и Fetcho поймёт куда её следует перенаправить
- **Вставка в один клик** -- кнопка Paste читает буфер обмена и заполняет поле ввода одним нажатием
- **Тёмная / Светлая тема** -- переключение сохраняется в `chrome.storage.local`; учитывает системные настройки
- **Минималистичный попап** -- крохотный popup с очень простым функционалом - ничего лишнего и отвлекающего

## Загрузка в браузер

1. Скачайте последний релиз Fetcho и распакуйте его
2. Откройте браузер и перейдите на `chrome://extensions` или `brave://extensions`
3. Включите **Режим разработчика** (переключатель в правом верхнем углу)
4. Нажмите **Загрузить распакованное расширение**
5. Выберите папку `dist/` из этого проекта
6. Иконка Fetcho появится на панели инструментов браузера

## Использование

1. Перейдите на любую поддерживаемую социальную сеть и скопируйте ссылку на видео
2. Нажмите на иконку Fetcho на панели инструментов
3. Нажмите кнопку **Paste**, чтобы вставить ссылку из буфера обмена, или вставьте её вручную
4. Попап определит платформу и покажет цветной badge
5. Нажмите **Download** — Fetcho откроет соответствующий сайт-загрузчик в новой вкладке
6. Контент-скрипт автоматически заполнит поле ввода и нажмёт кнопку загрузки; начнётся скачивание файла

> [!TIP]
> Если кнопка Paste не сработала, вы также можете вставить ссылку вручную (Ctrl+V / Cmd+V).

## Начало работы

### Требования

- Node.js 18 или выше
- Браузер на базе Chromium (Chrome, Edge, Brave, Opera и др.)

### Установка

```bash
# Клонирование репозитория
git clone https://github.com/your-username/fetcho.git
cd fetcho

# Установка зависимостей
npm install

# Сборка расширения
npm run build
```

Собранное расширение находится в директории `dist/`.

### Автоматизация контент-скриптов

Для каждого сайта-загрузчика существует отдельный контент-скрипт (например, `igram.ts` для Instagram). Каждый скрипт вызывает общую фабричную функцию `runContentScript()`, а затем работает с DOM конкретного сервиса:

```typescript
// Пример: src/content/igram.ts
import { runContentScript } from "./content-factory";

runContentScript({
  platformId: "instagram",
  waitSelector: 'input[type="text"], input[type="url"], #url',
  inputSelectors: ["#url", 'input[type="text"]', 'input[type="url"]'],
  buttonPattern: /download|submit|search|get/i,
  siteName: "igram.world",
});
```

Фабричная функция:

1. Читает `downloadUrl` и `platformId` из `chrome.storage.local`
2. Очищает хранилище, чтобы предотвратить повторный запуск
3. Ожидает появления поля ввода (до 5 секунд)
4. Устанавливает значение поля через нативный setter для корректной работы фреймворков
5. Генерирует DOM-события `input` и `change`
6. Ждёт 500 мс, затем находит и нажимает кнопку загрузки

Этот подход позволяет держать каждый контент-скрипт в пределах 10 строк и легко добавлять поддержку новых сайтов-загрузчиков.

## Конфигурация

Вся конфигурация находится в `vite.config.ts`, где объявляется манифест расширения:

- **Content script matches** — измените массив `content_scripts`, чтобы добавить или изменить правила внедрения скриптов
- **Host permissions** — измените массив `host_permissions`, чтобы предоставить доступ к дополнительным доменам
- **Определения платформ** — добавьте или отредактируйте записи в `src/shared/platforms.ts` для поддержки новых соцсетей

Настройка темы автоматически сохраняется в `chrome.storage.local` по ключу `theme`.

## Разрешения

| Разрешение | Обоснование |
|---|---|
| `clipboardRead` | Чтение буфера обмена при нажатии кнопки Paste |
| `storage` | Передача URL и ID платформы из попапа в контент-скрипты |
| `tabs` | Открытие сайта-загрузчика в новой вкладке браузера |

> [!IMPORTANT]
> Fetcho использует минимальный набор разрешений, необходимых для работы. Расширение не выполняет самостоятельных сетевых запросов — все данные передаются через стандартные API Chrome и пользовательские жесты.

## Разработка

```bash
# Запуск dev-сервера Vite с HMR
npm run dev

# Проверка типов и сборка для продакшена
npm run build

# Предпросмотр собранной версии
npm run preview
```

Во время разработки запустите `npm run dev` и загрузите папку `dist/` как распакованное расширение. Перезагружайте расширение из `chrome://extensions/` после каждой сборки. Плагин CRXJS Vite поддерживает HMR для попапа и контент-скриптов.

### Добавление новой платформы

1. Добавьте определение платформы в `src/shared/platforms.ts` (шаблон URL, целевой URL, цвет badge)
2. Создайте контент-скрипт в `src/content/`, который вызывает `runContentScript()` с нужными селекторами
3. Зарегистрируйте новый content script match и host permission в `vite.config.ts`
4. Пересоберите и перезагрузите расширение
